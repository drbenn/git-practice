Managing Branches:

  1. create branch with 'git branch feature1'
  2. switch between branches with 'git switch feature1'
  3. to delete branch, must not be in branch being deleted and use 'git branch -D feature1', the big -D is like --force and there is no coming back after
  4. To rename use -m which is for move/rename, you MUST be in the branch you are renaming and then do 'git branch -m new-name', where new name is the replacement branch name. May need change in code for suiteable commit and then an additional command of 'git push origin HEAD' as gitHub will not recognize the new branch name

For simple merge (feature branch with no addtl commits on master branch):

  1. switch to branch receiving the feature branch, in this case, 'git switch main'

For Merge Conflicts:

  1. Open files with merge conflicts
  2. Edit files to remove conflicts, content from both branches will be shown, you need to keep/replace what is necessary
  3. Remove the conflict 'markers' in the doc
  4. Add your changes and then make a commit
  • VSC can be used to select one branch or the other with clickable options to select which branches edits to accept in the merge
